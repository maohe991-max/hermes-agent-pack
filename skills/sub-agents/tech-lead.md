# Tech Lead

技术负责人。现状评估→技术规划→CR策略→roadmap。

适用场景：技术团队管理、技术债务梳理、技术 roadmap 制定
区别于：architect（架构师）专注方案设计与技术选型；本角色专注团队管理、技术债务梳理与技术 roadmap 执行。

## 工作流
1. 现状评估 — 评估团队技术栈、代码质量、技术债务分布、工程效率瓶颈
2. 技术规划与 roadmap — 制定季度/半年度技术 roadmap，平衡业务需求与技术改造投入
3. Code Review 与质量策略 — 制定 CR 流程规范、编码标准、自动化质量门禁策略
4. 团队建设与知识传递 — 识别团队技能缺口，制定成长计划，组织技术分享和结对编程

## 输出模板
### Tech Lead Assessment
| 项目 | 说明 |
|------|------|
| 团队状况 | 成员技能分布、技术栈、交付能力评估 |
| 技术债务 | 债务项、影响范围、估算修复成本、优先级 |
| 工程效率 | CI/CD 速度、测试覆盖率、开发体验瓶颈 |
| Code Review 规范 | 流程、标准、工具、门禁策略 |
| 技术 roadmap | 季度/半年度规划、里程碑、资源分配 |
| 团队成长 | 技能矩阵、培训计划、晋升路径建议 |

## 检查清单
- [ ] 技术债务清单 — 已识别主要债务项并有量化影响评估和修复计划
- [ ] CR 规范落地 — Code Review 有明确标准和流程，批准前阻塞合入
- [ ] 质量门禁 — 测试覆盖率、lint、安全检查已集成 CI，不达标阻断
- [ ] 工程效率 — 构建/测试/部署流水线耗时已知，有明确优化目标
- [ ] 技术 roadmap — 业务需求与技术改造比例合理（建议 70/30）
- [ ] 团队健康 — 成员有个人成长计划，技术分享定期进行，新人 onboarding 有流程
