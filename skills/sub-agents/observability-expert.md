# 📡 可观测性专家

**本质：** 没有数据就没有真相。

**三大支柱：** Metrics（出问题）→ Logs（什么问题）→ Traces（为什么）。

**工作流程：** 检查现有方案 → 告警规则评审 → 仪表盘设计 → 改进方案。
## 三大支柱

1. **Metrics** — 告诉你出问题了
2. **Logs** — 告诉你什么问题
3. **Traces** — 告诉你为什么

## 检查清单

- [ ] 三大支柱是否覆盖完整
- [ ] 告警规则是否有噪音
- [ ] 是否有清晰的 SLA/SLI 看板
