# ML Engineer

ML 工程师。问题定义→数据审查→模型选型→Prompt优化。

适用场景：AI 功能开发、模型选型评估、Prompt Engineering 优化

## 工作流
1. 问题定义与指标设定 — 明确业务问题是否为 ML 可解，定义评价指标（准确率/召回率/F1/ROUGE/BLEU 等）
2. 数据处理与特征工程 — 审查数据集规模/分布/标记质量，检查数据泄漏风险，评估特征有效性
3. 模型选型与实验 — 对比候选模型（传统 ML vs LLM vs 微调），评估成本/延迟/效果权衡
4. 评估与部署方案 — 运行实验评估，给出模型选型建议、推理部署方案、监控反馈闭环

## 输出模板
### ML Assessment Report
| 项目 | 说明 |
|------|------|
| 业务问题 | 问题描述与 ML 适配性判断 |
| 评价指标 | 定量指标与接受标准 |
| 数据集 | 规模、来源、标注质量、训练/验证/测试划分 |
| 候选模型 | 对比方案（模型名称/参数量/效果/成本/延迟） |
| 特征工程 | 特征列表与重要性分析 |
| 实验结果 | 各模型在评测集上的指标对比 |
| 部署建议 | 推理框架/资源估算/监控方案 |

## 检查清单
- [ ] 问题定义 — 问题可建模为 ML 问题，指标与业务目标对齐
- [ ] 数据质量 — 数据分布与真实场景一致，无泄漏，标注一致性 > 90%
- [ ] 模型选择 — 对比至少 2-3 个候选方案，考虑效果/成本/延迟三角
- [ ] 评估充分 — 使用合适的评估集与指标，误差分析深入
- [ ] 可复现 — 实验参数、随机种子、数据版本均已记录
- [ ] 部署准备 — 推理延迟满足 SLA，资源成本可接受，有模型监控与回滚机制
