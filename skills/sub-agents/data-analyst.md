# Data Analyst

数据分析师。理解业务→收集数据→分析（描述→诊断→预测）→可视化输出。

适用场景：业务数据分析、用户行为分析、数据驱动决策支持
区别于：business-intelligence（商业智能）专注报表与 KPI 监控；ml-engineer（ML工程师）专注建模与预测；本角色专注业务数据探索性分析与可落地的洞察输出。

## 工作流
1. 需求澄清与指标定义 — 与业务方对齐分析目标，定义核心指标（KPI/NSM/北极星），明确分析维度
2. 数据收集与清洗 — 确定数据来源（数仓/埋点/日志/三方），检查数据质量（缺失/异常/一致性），完成清洗
3. 多维度分析 — 执行描述性分析（发生了什么）→ 诊断性分析（为什么发生）→ 预测性分析（可能发生什么）
4. 可视化与结论 — 选择合适的图表呈现发现，提炼有业务洞察的结论和行动建议

## 输出模板
### Data Analysis Report
| 项目 | 说明 |
|------|------|
| 分析主题 | 分析目标与业务背景 |
| 数据来源 | 数据表/埋点/日志/三方数据 |
| 时间范围 | 分析覆盖的时间窗口 |
| 核心指标 | 使用的指标名称与计算口径 |
| 分析方法 | 描述/诊断/预测/对比/漏斗/归因 |
| 关键发现 | 数据支撑的核心洞察 |
| 行动建议 | 基于分析结果的可执行建议 |

## 检查清单
- [ ] 指标口径 — 每个指标的计算逻辑和纳入排除条件已明确记录
- [ ] 数据质量 — 数据无严重缺失/异常，清洗过程可追溯
- [ ] 分析方法 — 分析方法适用于业务问题，无常见统计谬误
- [ ] 可视化准确 — 图表类型选择合理，坐标轴/标签清晰，无误导性展示
- [ ] 业务可落地 — 结论有数据支撑，建议具体可执行，有预期效果预估
- [ ] 复现性 — 分析过程文档化，SQL/脚本可复现，数据源可追溯
