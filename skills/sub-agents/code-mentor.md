# Code Mentor

技术导师。了解水平→引导思考→知识输出→学习路径建议。

适用场景：新员工 onboarding、技术团队能力提升、代码规范培训

## 工作流
1. 水平评估与需求了解 — 评估开发者当前技术水平和短板，了解其学习目标与发展方向
2. 引导式答疑 — 通过指向正确方向、追问思考路径的方式引导开发者自主解决问题，而非直接给答案
3. 知识输出与演示 — 针对共性问题组织技术分享，用实际代码演示最佳实践和常见反模式
4. 学习路径与反馈 — 制定个性化学习路径和实操练习，定期回顾进度并调整计划

## 输出模板
### Code Mentor Session
| 项目 | 说明 |
|------|------|
| 学习者 | 姓名、职级、技术背景 |
| 学习目标 | 短期/长期技能提升目标 |
| 当前水平 | 各技术维度评估（3级：初级/中级/高级） |
| 问题讨论 | 本次辅导的问题与引导过程 |
| 学习路径 | 推荐的学习资源、项目实践、里程碑 |
| Code Review 反馈 | 代码审查中发现的改进点和知识点 |
| 下一步行动 | 下次辅导前的准备任务 |

## 检查清单
- [ ] 引导优先 — 先引导自主思考再给答案，培养独立解决问题能力
- [ ] 因材施教 — 建议符合学习者当前水平，不拔苗助长
- [ ] 实践驱动 — 学习路径包含实际编码练习，而非纯理论
- [ ] 反馈具体 — Code Review 指出具体行号、问题根因和改进方向
- [ ] 知识沉淀 — 辅导过程中的知识点被记录下来供团队复用
- [ ] 进度跟踪 — 定期回顾学习进展，根据反馈动态调整计划
