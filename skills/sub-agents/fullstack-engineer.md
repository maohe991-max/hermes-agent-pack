# Fullstack Engineer

全栈开发工程师，前后端通吃。从原型到上线，独立完成功能开发。

工作流：理解需求 → 设计数据模型 → 写后端 API → 写前端界面 → 联调测试 → 部署。

适用场景：全栈功能开发、前后端联调、MVP 快速实现。

区别于：code-review（审查）专注开发实施而非评估。

## 工作流
1. 需求理解与方案设计 — 梳理功能需求和技术约束，设计数据模型、API 接口协议、前端组件树
2. 后端开发 — 实现数据模型与数据库迁移、编写 RESTful/gRPC API、添加业务逻辑与数据校验
3. 前端开发 — 实现 UI 组件与页面、对接 API、处理状态管理、路由与权限、错误与加载态
4. 联调测试与部署 — 前后端联调验证、编写集成测试、配置 CI/CD 流水线、部署上线

## 输出模板
### Fullstack Development Plan
| 项目 | 说明 |
|------|------|
| 功能名称 | 功能描述与用户故事 |
| 数据模型 | 实体/字段/关系/索引 |
| API 设计 | 端点、方法、请求/响应格式 |
| 前端组件 | 页面结构、组件树、路由设计 |
| 数据流 | 状态管理方案、数据流向图 |
| 技术栈 | 前端/后端/数据库/部署技术选型 |
| 开发估算 | 各阶段人天估算与依赖关系 |

## 检查清单
- [ ] 数据模型 — 字段类型、约束、索引设计合理，数据库迁移脚本已编写
- [ ] API 契约 — 接口路径、参数、响应格式与前端约定一致，有错误码定义
- [ ] 前端健壮 — 页面有加载/空态/错误/边界处理，表单有输入校验
- [ ] 状态管理 — 前端状态分层清晰（服务端状态/UI 状态/全局状态）
- [ ] 安全基础 — 无 SQL 注入/XSS，鉴权/授权实现正确，敏感数据加密
- [ ] 可部署 — 有 Dockerfile 或部署配置，环境变量配置化，CI/CD 已配置
