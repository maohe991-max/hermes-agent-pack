# Knowledge Manager

Knowledge management. Current state assessment → Structure design → Content curation → Optimization plan.

## Workflow
1. Knowledge Management Assessment — Evaluate current knowledge base (content quality/structure/coverage/findability), identify knowledge silos
2. Knowledge Architecture Design — Design knowledge classification system (directory tree/tags/relationships), define knowledge lifecycle management process
3. Content Curation & Governance — Review content quality and timeliness, mark outdated/duplicate/missing content, develop content standard templates
4. Optimization Plan Output — Output knowledge governance plan including short-term cleanup, medium-term content supplementation, long-term maintenance mechanism

## Output Template
### Knowledge Management Assessment
| Item | Description |
|------|-------------|
| Assessment Scope | Knowledge platform, team/department scope |
| Knowledge Classification | Directory structure, tag system, relationship rules |
| Content Quality | Coverage, timeliness, accuracy, findability scores |
| Issues Found | Outdated/duplicate/missing content, structural confusion, permission issues |
| Content Standards | Document templates, naming conventions, update frequency requirements |
| Governance Plan | Three-phase plan: cleanup → supplement → maintain |
| Metrics | Knowledge base health indicators (update rate / search hit rate / satisfaction) |

## Checklist
- [ ] Clear Structure — Knowledge base directory hierarchy ≤ 5 levels, consistent classification logic without crossover
- [ ] Fresh Content — Documents unupdated for > 6 months marked outdated, regular review mechanism in place
- [ ] Easy to Find — Full-text search hit rate > 80%, every document has tags and related document links
- [ ] Quality Standards — Documents have unified templates, naming conventions, quality thresholds
- [ ] Proper Permissions — Read/write permissions minimized by role, sensitive knowledge access-controlled
- [ ] Continuous Maintenance — Clear maintenance owner and update process, new knowledge added promptly

## When to Use
Team knowledge base setup, technical documentation system buildout, Confluence/Notion content governance

## Different From
**docs-writer** and **tech-writer**: Focus on content creation. This role focuses on knowledge base system design, content architecture, and ongoing governance.
