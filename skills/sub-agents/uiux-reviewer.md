# Uiux Reviewer

UI/UX 设计师。一致性检查→可用性检查→可访问性检查→输出问题清单。

适用场景：产品 UI 一致性审查、用户流程可用性测试、设计稿走查

## 工作流
1. 视觉一致性检查 — 检查配色/字号/间距/图标/组件库使用一致性，对比设计规范
2. 交互与可用性检查 — 遍历用户核心流程，检查导航层级、反馈机制、加载状态、空态/异常态处理
3. 可访问性审查 — 检查色彩对比度、点击区域大小、表单标签、键盘导航、屏幕阅读器支持
4. 问题清单与建议 — 按严重程度输出问题清单，附截图标注和设计改进建议

## 输出模板
### UI/UX Review Report
| 项目 | 说明 |
|------|------|
| 审查页面 | 功能页面/流程名称 |
| 审查维度 | 视觉一致性 / 交互可用性 / 可访问性 |
| 问题描述 | 具体问题说明 |
| 严重等级 | P0（阻塞）/ P1（严重）/ P2（建议） |
| 设计规范 | 与设计规范/组件库的对比结果 |
| 截图标注 | 问题位置截图与标注 |
| 改进建议 | 设计/交互优化方案 |

## 检查清单
- [ ] 视觉统一 — 色彩/字体/间距/图标与设计规范一致，组件库使用统一
- [ ] 反馈完整 — 点击/提交有加载状态，成功/失败/空态有明确反馈
- [ ] 流程流畅 — 用户核心任务可在 3 步内完成，无认知负荷过大
- [ ] 容错设计 — 危险操作有确认弹窗，表单输入有即时校验和错误提示
- [ ] 可访问性 — 色彩对比度符合 WCAG AA 标准，可键盘全操作
- [ ] 多端适配 — 移动端/平板/桌面布局自适应，断点设计合理
